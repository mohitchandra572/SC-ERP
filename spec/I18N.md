# Internationalization (i18n) Specification

## 1. Strategy
- **Default Language**: Bangla (`bn`).
- **Support**: Bangla (`bn`) and English (`en`).
- **Library**: `next-intl` or simple React Context + JSON, depending on complexity. Given Phase 0 simplicity and "NO hard-coded strings", a lightweight custom hook or `next-intl` is suitable. We will use a lightweight Context approach for Phase 0 to avoid heavy dependencies if possible, or standard `next-intl` if requested (not explicitly requested, but standard for Next.js).
- **Format**: JSON key-value pairs.

## 2. Storage
- **Public Users**: Browsing session / LocalStorage / Cookie (`NEXT_LOCALE`).
- **Authenticated Users**: Persisted in `UserProfile.language` database field.
- **Middleware**: Detects locale from Cookie -> DB (if logged in logic usually complex in middleware, simplified: Middleware checks cookie, Client Component syncs DB preference on load).

## 3. Implementation Details
### Translation Files
- `src/locales/bn.json`
- `src/locales/en.json`

### Structure
```json
{
  "common": {
    "login": "লগইন",
    "submit": "জমা দিন"
  },
  "nav": {
    "home": "হোম",
    "about": "আমাদের কথা"
  }
}
```

### UI Components
- **LanguageSwitcher**: A persistent utility in Header/Navbar.
- **Hook**: `useTranslation()` returning the translation function `t('key')`.

## 4. Requirements
- No hardcoded text in TSX files.
- Fallback to English if Bangla key missing (or vice versa? Default is Bangla, so fallback might be Bangla).
- Dynamic data (like database content) is NOT translated by this system, only UI Static Text.
